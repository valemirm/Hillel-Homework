8. Баги: що дуже корисне відсутнє у наступному оголошенні класу?
- відсутній сеттер, наприклад SetAge(int Age);, який дозволив би задати значення змінної itsAge
- також метод GetAge() належить до категорії private за замовчуванням, оскільки не було об'явлено його у public, в результаті чого його не можна буде викликати;

class Cat
{
    int GetAge() const;

private:
    int itsAge;
};

9. Баги: які три помилки виявить компілятор у цьому коді?
- myTV.itsStation = 9; - itsStation належить до private категорії, тому не можна задати значення так напряму
- TV.SetStation(10) - не можна надавати значення самому типу даних, тобто класу TV; можна надавати дані лише об'єктам класу, тому треба записати як myTv.SetStation(10)
- TV myOtherTV(2); - у налаштуваннях класу відсутнїй конструктор, а стандартний не приймає жодних початкових значень

- не баг, а спостереження: наскільки мені відомо, то небажано лишати main без вказання його типу даних (int) та без return;


class TV
{
public:
    void SetStation(int Station);
    int GetStation() const;

private:
    int itsStation;
};

main()
{
    TV myTV;
    myTV.itsStation = 9;
    TV.SetStation(10);
    TV myOtherTV(2);
}
